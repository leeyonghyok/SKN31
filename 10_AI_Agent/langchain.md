# LangChain이란?

LangChain은 **대규모 언어 모델(LLM, Large Language Model)** 을 활용한 애플리케이션을 더 쉽게 만들 수 있도록 도와주는 오픈소스 프레임워크입니다. 단순히 모델에 프롬프트를 보내는 수준을 넘어서, 여러 구성요소를 연결해 복잡한 AI 워크플로를 만들 수 있게 해 줍니다.

## 핵심 목적

LangChain의 가장 큰 목적은 LLM 기반 애플리케이션 개발을 **표준화하고, 연결하고, 확장하기 쉽게 만드는 것**입니다. 이를 위해 다음과 같은 문제를 해결합니다.

- 프롬프트 작성과 관리
- 외부 데이터와 모델 연결
- 여러 단계의 작업 체인 구성
- 도구 호출 및 에이전트 구현
- 메모리와 대화 상태 관리
- 검색 증강 생성(RAG) 구현
- 다양한 모델/벡터DB/데이터 소스와의 통합

## LangChain의 주요 개념

### 1. Prompt Templates

LLM에 전달할 프롬프트를 템플릿 형태로 관리할 수 있습니다. 같은 구조의 프롬프트를 재사용하면서 입력값만 바꿔 넣을 수 있어 유지보수가 쉬워집니다.

예:
- 사용자 질문
- 시스템 지시문
- 출력 형식 지정

### 2. Chains

여러 작업을 순서대로 연결한 파이프라인입니다. 예를 들어:

1. 사용자 질문 입력
2. 관련 문서 검색
3. 검색 결과를 프롬프트에 삽입
4. LLM이 답변 생성

이처럼 하나의 단순 호출이 아니라, 여러 단계를 묶어 하나의 흐름으로 만들 수 있습니다.

### 3. Agents

에이전트는 LLM이 상황에 따라 **어떤 도구를 사용할지 스스로 선택**하도록 하는 방식입니다. 예를 들어:

- 검색 엔진 사용
- 계산기 사용
- DB 조회
- API 호출

즉, 모델이 단순히 답만 내는 것이 아니라, 필요한 행동을 결정하고 수행할 수 있게 합니다.

### 4. Tools

에이전트가 사용할 수 있는 외부 기능입니다. 대표적으로:

- 웹 검색
- 데이터베이스 쿼리
- 계산
- 파일 읽기/쓰기
- 외부 API 연동

### 5. Memory

대화형 애플리케이션에서 이전 대화 내용을 기억하도록 하는 기능입니다. 이를 통해 챗봇이 문맥을 유지할 수 있습니다.

예:
- 직전 질문과 답변 기억
- 사용자 선호도 저장
- 장기 대화 맥락 유지

### 6. Retrievers

문서나 데이터에서 관련 정보를 찾아오는 구성요소입니다. 주로 RAG(검색 증강 생성)에서 사용됩니다.

예:
- 벡터DB에서 유사 문서 검색
- 키워드 기반 문서 검색
- 하이브리드 검색

### 7. LCEL(LangChain Expression Language)

LangChain은 최근에 표현식 기반의 구성 방식을 강조합니다. 이를 통해 여러 컴포넌트를 더 선언적이고 간단하게 연결할 수 있습니다.

장점:
- 코드 가독성 향상
- 체인 구성 간소화
- 비동기/스트리밍 처리 지원

## LangChain으로 만들 수 있는 것

LangChain은 다음과 같은 앱 개발에 자주 사용됩니다.

- 챗봇
- 문서 QA 시스템
- 사내 지식 검색 도구
- 요약 서비스
- 데이터 분석 보조 에이전트
- 업무 자동화 워크플로
- RAG 기반 고객 지원 시스템

## LangChain의 장점

### 1. 빠른 개발

복잡한 LLM 앱에 필요한 기능들을 조합형으로 제공해 개발 속도를 높여줍니다.

### 2. 다양한 통합

OpenAI, Anthropic, Hugging Face 같은 모델뿐 아니라 벡터 DB, 검색 엔진, API 등과의 연결이 쉽습니다.

### 3. 모듈화

기능이 잘 분리되어 있어 필요한 부분만 선택해 사용할 수 있습니다.

### 4. RAG 구현에 강함

문서 검색과 답변 생성을 자연스럽게 연결하기 좋아, 지식 기반 챗봇에 특히 유용합니다.

## 단점 또는 주의점

LangChain은 강력하지만, 모든 경우에 최선은 아닙니다.

- 구조가 복잡해질 수 있음
- 버전 변화가 빠른 편이라 학습 비용이 있음
- 작은 프로젝트에서는 오히려 과할 수 있음
- 추상화가 많아 내부 동작을 이해하기 어려울 수 있음

## 언제 사용하면 좋은가?

LangChain은 다음 상황에서 특히 유리합니다.

- LLM을 단순 호출하는 것 이상이 필요할 때
- 여러 데이터 소스와 연결해야 할 때
- 검색 기반 답변(RAG)을 구현할 때
- 에이전트처럼 도구 사용이 필요할 때
- 챗봇에 기억 기능을 넣고 싶을 때

## 간단한 예시 개념

예를 들어 사용자가 “이 문서 요약해줘”라고 요청하면 LangChain은 다음 흐름을 구성할 수 있습니다.

1. 문서를 로드한다.
2. 문서를 적절한 크기로 나눈다.
3. 관련 내용을 검색한다.
4. 프롬프트를 만든다.
5. LLM에 전달해 요약을 생성한다.

이 과정을 하나의 애플리케이션 흐름으로 연결하는 것이 LangChain의 강점입니다.

## 요약

LangChain은 **LLM 애플리케이션을 만들기 위한 프레임워크**로, 프롬프트, 체인, 에이전트, 도구, 메모리, 검색 기능 등을 조합해 복잡한 AI 시스템을 쉽게 구축할 수 있게 해 줍니다. 특히 RAG와 에이전트 기반 애플리케이션을 만들 때 매우 유용합니다.
