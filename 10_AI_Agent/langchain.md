# LangChain이란?

LangChain은 대규모 언어 모델(LLM, Large Language Model)을 활용한 애플리케이션을 더 쉽게 만들기 위한 프레임워크입니다. 단순히 모델에 질문을 보내고 답을 받는 수준을 넘어서, 여러 단계의 작업, 외부 도구 연동, 데이터 검색, 메모리 유지, 에이전트 구성 등을 체계적으로 연결할 수 있게 도와줍니다.

## 핵심 목적

LLM 기반 애플리케이션은 보통 다음과 같은 요소를 함께 다룹니다.

- 프롬프트 설계
- 모델 호출
- 검색(RAG)
- 외부 API 또는 도구 호출
- 대화 상태 및 메모리 관리
- 여러 단계의 작업 흐름(워크플로우)
- 에이전트 기반 의사결정

LangChain은 이런 요소들을 하나의 일관된 방식으로 묶어서 개발자가 복잡한 LLM 애플리케이션을 더 빠르고 구조적으로 만들 수 있도록 설계되었습니다.

## 주요 특징

### 1. 체인(Chains)

여러 작업 단계를 순차적으로 연결하는 구조입니다. 예를 들어:

- 사용자 질문을 받아 프롬프트를 생성하고
- 모델 응답을 받은 뒤
- 후처리를 수행하는 흐름

이런 과정을 체인으로 구성할 수 있습니다.

### 2. 에이전트(Agents)

에이전트는 LLM이 상황에 따라 어떤 도구를 사용할지 스스로 판단하도록 만드는 기능입니다. 예를 들어 검색 API, 계산기, 데이터베이스 조회 도구 등을 필요에 따라 선택하게 할 수 있습니다.

### 3. 도구(Tools)

LangChain은 외부 기능을 도구로 연결하는 것을 지원합니다. 예를 들면:

- 웹 검색
- DB 조회
- 파일 읽기/쓰기
- 계산
- 사내 시스템 API 호출

이를 통해 LLM이 단순 생성 모델이 아니라 실제 작업을 수행하는 컴포넌트처럼 동작할 수 있습니다.

### 4. RAG(Retrieval-Augmented Generation)

LangChain은 검색 증강 생성(RAG) 구현에 많이 사용됩니다. RAG는 모델이 답을 생성하기 전에 관련 문서를 검색하여, 더 정확하고 최신 정보를 바탕으로 응답하도록 돕는 방식입니다.

일반적인 흐름은 다음과 같습니다.

1. 문서를 분할(chunking)
2. 임베딩 생성
3. 벡터 저장소에 저장
4. 질문이 들어오면 관련 문서 검색
5. 검색 결과를 프롬프트에 넣어 답변 생성

### 5. 메모리(Memory)

대화형 애플리케이션에서는 이전 대화 내용을 기억해야 하는 경우가 많습니다. LangChain은 대화 기록이나 상태를 유지하는 기능을 제공하여, 챗봇이 문맥을 이어갈 수 있게 합니다.

### 6. 다양한 통합(Integrations)

LangChain은 많은 LLM 제공자와 데이터 소스, 벡터 데이터베이스, 검색 엔진, 클라우드 서비스와 연동할 수 있도록 설계되어 있습니다.

예시:

- OpenAI
- Anthropic
- Google
- Hugging Face
- Pinecone
- FAISS
- Chroma
- Elasticsearch
- PostgreSQL

## LangChain의 구성 요소

LangChain은 크게 다음과 같은 개념으로 이해할 수 있습니다.

- **Prompt Templates**: 프롬프트를 재사용 가능하게 관리
- **Models / LLMs**: 텍스트 생성 모델 호출
- **Chains**: 여러 작업을 연결
- **Agents**: 도구 사용을 자동으로 결정
- **Tools**: 외부 기능 연결
- **Retrievers**: 관련 문서 검색
- **Memory**: 대화 상태 유지
- **Callbacks**: 실행 과정 추적 및 로깅

## 장점

- 복잡한 LLM 애플리케이션을 구조화하기 좋음
- 다양한 서비스와 쉽게 통합 가능
- RAG, 챗봇, 에이전트 구현에 유용
- 프로토타입을 빠르게 만들 수 있음

## 단점 및 주의점

- 추상화가 많아 처음에는 구조가 복잡하게 느껴질 수 있음
- 버전 변화가 비교적 잦아 문서와 코드가 달라질 수 있음
- 너무 많은 기능을 한 번에 쓰면 디버깅이 어려울 수 있음

## 언제 쓰면 좋은가?

LangChain은 다음과 같은 상황에서 특히 유용합니다.

- LLM 기반 챗봇을 만들 때
- 사내 문서 검색 시스템(RAG)을 만들 때
- LLM이 외부 도구를 사용해야 할 때
- 여러 단계의 작업을 자동화할 때
- 에이전트 형태의 AI 서비스를 구현할 때

## 간단한 예시 개념

예를 들어, 사용자가 "우리 회사 환불 정책이 뭐야?"라고 물으면:

1. LangChain이 질문을 받아 관련 문서를 검색하고
2. 검색한 문서를 프롬프트에 포함한 뒤
3. LLM이 그 문서를 바탕으로 답변을 생성합니다.

이렇게 하면 모델이 학습하지 않은 최신 사내 정보도 활용할 수 있습니다.

## 정리

LangChain은 LLM을 활용한 애플리케이션 개발을 위한 프레임워크로, 프롬프트, 체인, 에이전트, 도구, 검색, 메모리 등을 연결해 복잡한 AI 시스템을 쉽게 구축하도록 돕습니다. 특히 RAG와 에이전트 기반 워크플로우를 구현할 때 많이 사용됩니다.
